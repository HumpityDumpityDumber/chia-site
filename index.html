Declarative Configuration

Chia is a Nix-like declarative configuration and runtime manager built exclusively for Arch Linux, bringing reproducible system management to the bleeding edge.

Understanding .chia Syntax

-- app-name --

Defines an isolated app environment. Each app must define exactly one package.

-- terminal --
alacritty
%auto
share^

package^ / binary^

Override package or binary names. The ^ directive goes on its own line, then the value on the next line.

visual-studio-code-bin
package^
code
binary^

%auto / %none

%auto includes pacman dependencies for the single package above. %none skips dependencies entirely.

discord-ptb
%auto

share^ / needs^

share^ makes other defined apps available to this one. needs^ defines dependencies on other apps. Both appear alone.

share^
needs^

Organized Configuration

Chia uses a structured approach to configuration management:

  • Entry Point: Main configuration file that ties everything together
  • Packages: Separate user and system package definitions
  • Configs: Application-specific configuration files
  • System: System-level settings and network configuration

Quick Examples

  -- alacritty --
  alacritty
  %auto

  needs^

  %auto
  share^
  needs^
  
  -- main browser --
  firefox
  %none 
  share^
  needs^
